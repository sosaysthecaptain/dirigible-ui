Loading Screen

A centered spinner with optional text label, used as a full-page or section-level loading state. This is the simplest loading pattern — an indeterminate spinner above a muted text label. For determinate progress, see Progress.

When to Use

  • Initial app load before content is available
  • Full-page transitions between sections
  • Content area placeholder while data loads
  • Any async state where you need a simple "please wait" indicator

Variants

Variant Description
Full-pageCentered vertically and horizontally, fills the viewport. 24px spinner + "Loading..." text.
Without textSpinner only, no label. Use when the context makes loading obvious.
Small / inline14px spinner with horizontal text layout. For inline or compact spaces.
Custom textSame layout but with a specific message like "Syncing notes..." or "Saving..."

Anatomy

<div class="loading">                            <!-- container -->
  <div class="loading-spinner" />                <!-- 24px indeterminate spinner -->
  <div class="loading-text">Loading...</div>    <!-- optional label -->
</div>

Do's and Don'ts

Do

  • Center the loading state both vertically and horizontally in its container
  • Use var(--muted) for text — loading states should be quiet
  • Use the 14px spinner for inline/compact loading states
  • Provide context-specific text when helpful ("Syncing notes...", "Saving...")

Don't

  • Don't use skeleton screens — this component is the simple spinner+text pattern
  • Don't use accent-colored spinners — loading is utilitarian, use var(--fg)
  • Don't show loading for operations under ~200ms — it causes flash
  • Don't combine with progress bars — use Progress when you have a percentage

Rules

  1. Default spinner is 24px with 3px solid var(--fg) border and transparent top.
  2. Text is "Loading..." by default, 13px, var(--muted), 12px below spinner.
  3. Full-page variant uses min-height: 100vh and centers with flexbox.
  4. Section-level variant uses the section's height instead of viewport.
  5. Inline variant uses horizontal layout with gap: 8px and 14px spinner.
  6. Animation uses @keyframes spin — same keyframes as Progress spinners.
