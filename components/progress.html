Progress Bar & Spinners

Three related loading indicators: a horizontal determinate progress bar, a circular determinate spinner, and a circular indeterminate spinner. All use var(--fg) as the primary fill color — spinners and progress bars are utilitarian, not accented.

When to Use

  • Progress bar: File uploads, sync operations, multi-step processes where percentage is known
  • Determinate spinner: Compact circular progress where space is limited (e.g., inline in a table row)
  • Indeterminate spinner: Any async operation where duration is unknown — API calls, loading data, processing

Variants

Variant Type Description
Progress barDeterminateHorizontal track + fill. Label above, percentage right-aligned.
Determinate spinnerDeterminateSVG circle with arc showing completion. 24px default.
Indeterminate spinnerIndeterminateBorder-based rotating circle. 14px / 24px / 32px sizes.
SVG spinnerIndeterminateSVG-based rotating arc for smoother rendering.

Do's and Don'ts

Do

  • Use var(--fg) for progress fill — keep it utilitarian
  • Show a label and percentage for progress bars when space allows
  • Use the 14px spinner inline with text (e.g., button loading state)
  • Animate progress bar transitions with transition: width 0.3s ease
  • Use square ends (no border-radius) on progress bars per Dirigible style

Don't

  • Don't use accent color for standard progress — reserve for primary CTAs only
  • Don't add border-radius to progress bars — square ends are intentional
  • Don't show a determinate bar/spinner when you don't know the actual progress
  • Don't use spinners larger than 32px — use the Loading pattern for full-page states

Rules

  1. Progress bars have no border-radius — square ends match Dirigible's structural aesthetic.
  2. All indicators use var(--fg) as primary color by default.
  3. Spinner animation is spin 1s linear infinite — consistent timing across all sizes.
  4. Determinate spinners rotate the SVG -90deg so the arc starts from the 12 o'clock position.
  5. Progress bar label sits above the track, percentage is right-aligned.
  6. Three size tiers for each indicator: small, default, large.
