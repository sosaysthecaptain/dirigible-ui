Menu

A dropdown menu for application actions. Bold labels, compact rows, optional icons and keyboard shortcut hints. Distinct from Context Menu, which is a right-click kebab menu for item-specific options — menus are action-oriented navigation for app-level commands.

When to Use

  • App-level action menus (File, Edit, View)
  • Toolbar dropdown menus
  • Action menus triggered by a button click
  • Any general-purpose command list that isn't tied to a specific item

Variants

Variant Description
Standard Text-only menu items with optional keyboard shortcuts
With icons 16x16 icon at the start of each row, 8px gap to label
With shortcuts Right-aligned keyboard shortcut hints in muted 12px text
Nested Submenu arrow on the right edge, opens adjacent panel on hover

Anatomy

<div class="menu">                          <!-- container -->
  <div class="menu-item">                   <!-- row -->
    <svg class="menu-icon" />               <!-- optional icon -->
    <span class="menu-label">Label</span>  <!-- bold label -->
    <span class="menu-shortcut">⌘N</span>  <!-- optional shortcut -->
  </div>
  <div class="menu-divider" />              <!-- divider -->
  <div class="menu-item menu-item-danger"> <!-- danger item -->
    ...
  </div>
</div>

Do's and Don'ts

Do

  • Use bold (600) labels — this is the early Mac distinction from context menus
  • Group related actions with dividers
  • Place destructive actions last, separated by a divider
  • Show keyboard shortcuts for common actions
  • Use 16x16 icons consistently — all items with icons or none

Don't

  • Don't use checkmarks — menus are for actions, not selection state (use Context Menu for that)
  • Don't mix icon and non-icon items in the same section
  • Don't nest more than one level deep
  • Don't use menus for navigation — use the Tab Bar or Sidebar

Rules

  1. Container gets border-radius: var(--radius-md); individual items have no border-radius.
  2. Font weight is 600 for all menu labels. This is what distinguishes menus from context menus visually.
  3. Icons are always var(--muted) except on danger items where they inherit var(--danger).
  4. Keyboard shortcuts are right-aligned, 12px, weight 400, var(--muted).
  5. Hover uses color inversion: background: var(--fg), color: var(--bg). Icons and shortcuts also invert.
  6. Disabled items use opacity: 0.4 and suppress hover styling.
  7. Danger items color both text and icon with var(--danger). Danger hover uses background: var(--danger) with white text.
  8. Menu opens below the trigger with 4px gap. Position with position: absolute.
  9. Minimum width: 200px. Content determines actual width.
